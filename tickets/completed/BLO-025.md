# BLO-025: Add planned/ tier to ticket lifecycle

## Objective

Introduce a `tickets/planned/` directory for refined, ready-to-activate tickets. The existing `tickets/backlog/` becomes an idea pool for rough ticket drafts. This separates "brainstorming ideas" from "curated, dependency-checked tickets ready to work."

## Context

The project is about to generate 30-40+ new ticket ideas covering P1/P2 spec tools, beyond-spec features, and distribution tasks. Mixing rough ideas with refined, dependency-resolved tickets in a single `backlog/` directory makes prioritization harder. A two-tier system (backlog → planned) keeps the pipeline clean.

## Acceptance Criteria

- [x] `tickets/planned/` directory exists
- [x] BLO-017 moved from `backlog/` to `planned/` via `git mv`, status updated to `planned`
- [x] `tickets/backlog/` remains as the idea pool (empty after BLO-017 move, with .gitkeep)
- [x] CLAUDE.md updated: directory layout, lifecycle flow, status enum, all workflow references
- [x] `tickets/TEMPLATE.md` status field includes `planned`
- [x] `Makefile` verify-ticket target checks `planned/` for duplicates and status mismatches
- [x] `.claude/hooks/verify-ticket-completion.sh` checks `planned/` for orphaned files
- [x] `make check` passes

## Technical Approach

1. Create `tickets/planned/` directory
2. `git mv tickets/backlog/BLO-017.md tickets/planned/BLO-017.md`
3. Update BLO-017 frontmatter: `status: planned`
4. Add `.gitkeep` to `tickets/backlog/` (now empty)
5. Update CLAUDE.md lifecycle: `backlog → planned → ongoing → completed/rejected`
6. Update TEMPLATE.md status enum
7. Update Makefile verify-ticket
8. Update hook script to check planned/ directory
9. Run `make check`
