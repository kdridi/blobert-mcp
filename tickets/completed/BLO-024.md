# BLO-024: Ticket completion automation and lifecycle guardrails

## Objective

Prevent recurring ticket lifecycle mistakes — orphaned backlog files not staged, project_state.md committed separately, unchecked criteria — by hardening CLAUDE.md instructions, adding a Claude Code `Stop` hook for automated verification, adding a `make verify-ticket` target, and enforcing `git mv` for ticket moves.

## Context

BLO-020 and BLO-021 both required manual cleanup commits because:
1. Old `tickets/backlog/BLO-XXX.md` was deleted on disk but not `git rm`'d, leaving orphaned deletions.
2. `project_state.md` was updated in a separate commit instead of being staged with the completion commit.
3. Files were deleted and recreated in new directories instead of using `git mv`.

These are process failures, not code bugs. The fixes are: clearer instructions (CLAUDE.md), automated safety net (Stop hook), CI defense (Makefile target), and enforcing `git mv`.

## Acceptance Criteria

- [x] CLAUDE.md "4. Complete" section replaced with explicit numbered checklist including `git mv` and staging steps.
- [x] CLAUDE.md "2. Activate" section updated to mandate `git mv` (not delete+recreate).
- [x] CLAUDE.md "5. Reject" section updated to mandate `git mv`.
- [x] `.claude/hooks/verify-ticket-completion.sh` created — Stop hook that inspects git state after each turn and warns about orphaned files, unstaged project_state.md, unchecked criteria.
- [x] `.claude/settings.json` created with Stop hook configuration.
- [x] `make verify-ticket` target added — validates no duplicate ticket files across directories, max 1 ongoing, no unchecked criteria in completed tickets.
- [x] `make verify-ticket` wired into `make check`.
- [x] All existing checks still pass (`make check`).

## Technical Approach

### A. CLAUDE.md changes

Replace the "4. Complete" section with a numbered protocol. Update "2. Activate" and "5. Reject" to say `git mv` explicitly.

### B. Stop hook

Shell script at `.claude/hooks/verify-ticket-completion.sh` that:
1. Checks `git diff --cached --name-only` for `tickets/completed/BLO-*.md`
2. If found, extracts ticket ID and verifies:
   - No orphaned file in backlog/ or ongoing/ (on disk, not staged for deletion)
   - `project_state.md` not modified-but-unstaged
   - All acceptance criteria checked in the completed ticket
3. Outputs advisory JSON description if problems found

### C. Makefile target

`make verify-ticket` checks filesystem-level integrity. Added to `make check` pipeline.

## Notes

- Stop hook is advisory (warns), not blocking — it fires after Claude's turn ends.
- Hook script uses only bash builtins + grep/awk/wc — no external dependencies.
- `git mv` is the canonical way to move tickets between directories. This preserves git history.
