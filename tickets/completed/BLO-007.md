# BLO-007: Community & governance documentation

## Objective

Add the standard GitHub community health files (`CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, `SECURITY.md`, `CODEOWNERS`) so the project meets GitHub's community standards and sets clear expectations for contributors.

## Context

GitHub surfaces missing community health files as warnings in the repository insights. The project has contributor workflow documentation in `CLAUDE.md`, but no standalone files for external contributors. These documents contain policy decisions (contact info, enforcement procedures, security disclosure process) that require human judgment.

## Acceptance Criteria

- [x] `CONTRIBUTING.md` exists with: prerequisites (Python 3.10+, uv, git), development setup instructions, the ticket-based workflow (referencing CLAUDE.md golden rule), code style conventions, commit message format (`BLO-XXX: description`), and PR process
- [x] `CODE_OF_CONDUCT.md` adopts the Contributor Covenant v2.1, customized with project contact information
- [x] `SECURITY.md` contains: supported versions table, vulnerability reporting instructions (private disclosure via GitHub security advisories or email), response timeline expectations, and a note that the project handles user-supplied ROM files as untrusted input
- [x] `CODEOWNERS` assigns the repository owner (`@kdridi`) as the default reviewer for all paths

## Technical Approach

1. **`CONTRIBUTING.md`** — Extract and expand from the contributing section in README.md. Reference the ticket system in CLAUDE.md. Include `uv`-based workflow details.

2. **`CODE_OF_CONDUCT.md`** — Contributor Covenant v2.1 boilerplate with project-specific contact information.

3. **`SECURITY.md`** — GitHub-standard format. Note that ROM files are user-supplied untrusted input — relevant for the PyBoy emulator interaction.

4. **`CODEOWNERS`** — Place at root or `.github/CODEOWNERS`. Format: `* @kdridi`.

## Dependencies

None — these are standalone documentation files.
