# BLO-073: Add roadmap.md convention to planned/ directory

## Objective

Schedule ~20 tickets from backlog to planned/, create a `roadmap.md` file inside `tickets/planned/` that lists all planned tickets in their intended execution order, and update CLAUDE.md to document this convention. Done when tickets are moved, roadmap.md exists with correct ordering, and CLAUDE.md references the roadmap convention.

## Context

The project has 47 backlog tickets and no planned tickets. The user wants to schedule ~20 tickets (all 16 P1 + 4 strategic P2) into `planned/` with a defined execution order. A `roadmap.md` file provides the authoritative sequence, and CLAUDE.md must document this convention so future sessions know to consult it.

## Acceptance Criteria

- [x] 20 tickets moved from `tickets/backlog/` to `tickets/planned/` via `git mv`
- [x] Each moved ticket has `status: planned` and updated timestamp in frontmatter
- [x] Each moved ticket has a log entry recording the planning event
- [x] `tickets/planned/roadmap.md` created with all 20 tickets listed in execution order
- [x] Roadmap respects dependency ordering (no ticket appears before its prerequisites)
- [x] CLAUDE.md updated to document the roadmap.md convention

## Technical Approach

1. Move 20 tickets via `git mv tickets/backlog/BLO-XXX.md tickets/planned/BLO-XXX.md`
2. Update frontmatter (`status: planned`, `updated` timestamp) and append log entry
3. Create `tickets/planned/roadmap.md` with ordered list
4. Update CLAUDE.md directory layout and lifecycle sections
