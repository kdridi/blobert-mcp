# BLO-034: Byte pattern search and string detection

## Objective

Implement two P1 search tools — `find_byte_pattern` and `find_strings` — that allow an LLM to locate byte sequences and text strings within a loaded ROM. This enables identification of known function signatures, DMA routines, dialogue text, debug messages, and other recognizable patterns without manually inspecting every address.

## Context

The spec (docs/mcp-spec.md §1) defines `find_byte_pattern` (P1) and `find_strings` (P1) as important tools for a complete analysis workflow. With 25 P0 tools already implemented including ROM reading (BLO-013), the infrastructure for reading ROM bytes is in place. These search tools build on that foundation to provide higher-level discovery capabilities.

## Acceptance Criteria

- [x] `domain/search.py` module exists with pure functions for pattern matching and string detection
- [x] `find_byte_pattern(pattern, start?, end?)` searches ROM bytes for a hex byte sequence with wildcard support
- [x] Pattern syntax: hex bytes separated by spaces, `??` for single-byte wildcard (e.g., `"CD ?? 40"` matches any `CALL` to `0x40xx`)
- [x] `find_byte_pattern` returns a list of match addresses, capped at 100 results per spec
- [x] `find_strings(min_length?, encoding?)` locates text strings in ROM
- [x] String detection supports ASCII encoding
- [x] String detection supports Game Boy custom character encoding
- [x] `find_strings` returns list of (address, decoded text), capped at 200 results per spec
- [x] Both tools respect the standard error format (`{"error": "CODE", "message": "..."}`) on failure
- [x] Domain module has no project imports (domain isolation rule)
- [x] MCP tool handlers are thin: validate input, read bytes, delegate to domain, format output
- [x] Tests written before implementation (TDD) for both pattern matching and string detection
- [x] Optional `start`/`end` parameters allow restricting the search range

## Technical Approach

### Domain layer (`domain/search.py`)

1. **`match_byte_pattern(data: bytes, pattern: str, start: int = 0, end: int | None = None, max_results: int = 100) -> list[int]`**
   - Parse pattern string into a list of (byte_value | None) where None represents `??` wildcard
   - Slide over the data range, checking each position against the pattern
   - Return list of offsets where the pattern matches

2. **`find_text_strings(data: bytes, min_length: int = 4, encoding: str = "ascii", max_results: int = 200) -> list[tuple[int, str]]`**
   - For ASCII: scan for runs of printable characters (0x20-0x7E) of at least `min_length`
   - For GB custom: use a character mapping table for common Game Boy text encodings
   - Return list of (offset, decoded_string) tuples

### Tool layer (`tools/`)

- `find_byte_pattern` tool: validate pattern syntax, read ROM bytes, call `match_byte_pattern`, format response
- `find_strings` tool: validate parameters, read ROM bytes, call `find_text_strings`, format response

### Tests (`tests/domain/test_search.py`)

- Pattern matching: exact match, wildcard match, no match, multiple matches, boundary conditions
- String detection: ASCII strings, min_length filtering, GB custom encoding, overlapping strings
- Edge cases: empty pattern, pattern longer than data, start/end range limiting
