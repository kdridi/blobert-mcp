# BLO-009: CI/CD pipeline, pre-commit hooks & developer automation

## Objective

Set up automated quality enforcement (GitHub Actions CI, pre-commit hooks) and developer convenience tooling (Makefile) so that code quality is maintained from the first line of source code onward.

## Context

Once BLO-001 creates the project skeleton with `pyproject.toml` and `src/`, the project needs automated linting, formatting, and testing. Without CI, regressions can reach `main` unchecked. Without pre-commit hooks, developers must remember to run quality checks manually. A Makefile provides discoverable, standardized commands for common tasks.

## Acceptance Criteria

- [x] `.github/workflows/ci.yml` exists with a GitHub Actions workflow that: triggers on push to `main` and on pull requests, sets up Python 3.10+ with `uv`, runs `uv sync`, runs `ruff check`, runs `ruff format --check`, and runs `pytest`
- [x] `.pre-commit-config.yaml` exists with hooks for: trailing whitespace removal, end-of-file fixer, YAML/TOML syntax check, `ruff check --fix`, and `ruff format`
- [x] `Makefile` exists with targets for: `setup` (uv sync + pre-commit install), `test` (pytest), `lint` (ruff check), `format` (ruff format), `check` (lint + format check + test), `clean` (remove build artifacts), `run` (uv run blobert-mcp), and `help` (list targets)
- [x] `pyproject.toml` includes a `[tool.ruff]` section with: `target-version = "py310"`, `line-length = 88`, and appropriate rule selections (E, F, W, I, UP)
- [x] CI workflow passes on the current codebase without false failures
- [x] Pre-commit hooks install with `pre-commit install` and run successfully on all tracked files

## Technical Approach

1. **CI workflow** — Single job on `ubuntu-latest`. Steps: checkout, setup-python, install uv, `uv sync`, `ruff check`, `ruff format --check`, `pytest`. Cache uv cache directory for faster runs. No matrix needed initially (single Python version).

2. **Pre-commit** — Use `pre-commit` framework with hooks from `pre-commit/pre-commit-hooks` (trailing-whitespace, end-of-file-fixer, check-yaml, check-toml) and `astral-sh/ruff-pre-commit` (ruff-check, ruff-format). Add `pre-commit` as a dev dependency in `pyproject.toml`.

3. **Makefile** — GNU Make with `.PHONY` declarations. Simple, composable targets. Include a `help` target using `grep` on Makefile comments for self-documentation.

4. **Ruff configuration** — Add `[tool.ruff]` to `pyproject.toml`: `target-version = "py310"`, `line-length = 88`, `select = ["E", "F", "W", "I", "UP"]`. This replaces the need for separate `.flake8`, `.pylintrc`, and `black` configuration.

## Dependencies
