# BLO-001: Project infrastructure & MCP server skeleton

## Objective

Set up the Python project structure, dependencies, and a minimal MCP server that starts, connects, and registers tool stubs. After this ticket, `uv run blobert-mcp` launches a working MCP server with no tools yet — but the wiring is complete and ready for tools to be added incrementally.

## Context

The project has a complete spec (`docs/mcp-spec.md`) and research (`docs/journal.md`) but zero source code. Every other ticket depends on this foundation: package config, project layout, MCP server entry point, and the emulator session wrapper that all tools will share.

This is the first ticket — it unblocks everything else.

## Acceptance Criteria

- [x] `pyproject.toml` exists with project metadata, Python >=3.10, and dependencies: `pyboy`, `mcp` (MCP SDK), `Pillow`
- [x] Project uses `uv` for package management; `uv sync` installs all dependencies
- [x] Source layout: `src/blobert_mcp/` package with `__init__.py`, `server.py`, `emulator.py`
- [x] `server.py` creates an MCP server instance, registers a health-check tool (`ping`), and exposes stdio transport
- [x] `emulator.py` contains an `EmulatorSession` class that wraps PyBoy lifecycle (init, load ROM, shutdown) — no tools yet, just the shared state object
- [x] Entry point configured: `uv run blobert-mcp` starts the MCP server
- [x] The server starts without errors when no ROM is loaded (graceful empty state)
- [x] Basic `tests/` directory with a smoke test that imports the package

## Technical Approach

1. **`pyproject.toml`** — Use `[project]` table with `[project.scripts]` for the entry point. Pin `pyboy>=2.0.0`, `mcp>=1.0.0`, `Pillow>=10.0.0`. Dev dependencies: `pytest`.

2. **`src/blobert_mcp/__init__.py`** — Package marker, export version.

3. **`src/blobert_mcp/emulator.py`** — `EmulatorSession` class:
   - Holds the `PyBoy` instance (nullable — None when no ROM loaded)
   - `load_rom(path, headless=True)` — creates PyBoy instance
   - `shutdown()` — cleans up
   - Holds shared state: save states dict, breakpoints dict, knowledge base reference
   - Singleton-ish: one session per server process

4. **`src/blobert_mcp/server.py`** — MCP server setup:
   - Create `Server` instance from MCP SDK
   - Register `ping` tool (returns `{"status": "ok", "rom_loaded": bool}`)
   - `main()` function that runs the stdio transport
   - The `EmulatorSession` instance is created here and passed to tool handlers via closure or context

5. **Entry point** — `[project.scripts] blobert-mcp = "blobert_mcp.server:main"`

6. **`tests/test_smoke.py`** — Import check, verify server can be instantiated.

## Notes

- PyBoy 2.x API: `PyBoy(rom_path, window="null")` for headless mode.
- The `EmulatorSession` design should anticipate that all 25 P0 tools will need access to it — keep the interface clean and extensible.
- Do NOT implement any Game Boy tools in this ticket. Only `ping` for connectivity verification.
