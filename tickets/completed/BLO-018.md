# BLO-018: Fix ruff lint errors breaking CI and document pre-commit workflow

## Objective

Fix 8 ruff lint errors (E501, F401, I001) across 4 files that are causing all CI runs on main to fail. Add CLAUDE.md guidance for pre-commit hook usage to prevent recurrence.

## Context

BLO-014, BLO-015, and BLO-016 introduced lint violations that were not caught locally because the pre-commit hook was never installed. The repo has `.pre-commit-config.yaml` and a `make setup` target but they were not used. CI has been red for the last 5 commits.

## Acceptance Criteria

- [x] `uv run ruff check .` passes with 0 errors
- [x] `uv run ruff format --check .` passes
- [x] `uv run pytest` passes
- [x] CI pipeline passes on main
- [x] CLAUDE.md documents pre-commit hook and `make check` workflow
- [x] Pre-commit hook installed locally

## Technical Approach

1. Fix lint errors in 4 files (remove unused imports, wrap long lines, sort imports)
2. Install pre-commit hook locally via `uv run pre-commit install`
3. Add linting conventions section to CLAUDE.md

None.
