# BLO-014: Memory reading & hardware status tools

## Objective

Implement the second tool layer: memory access and hardware status tools. After this ticket, 4 additional MCP tools are registered — an LLM can read arbitrary memory (RAM, VRAM, I/O), access any ROM/RAM bank, query MBC bank state, and inspect interrupt flags.

## Context

This is the third of three tickets decomposed from BLO-002. It builds `tools/memory.py` on top of the domain layer completed in BLO-012. These tools complement the session/static tools (BLO-013) by providing dynamic memory inspection and hardware state access.

Covers spec sections: §1 (Static analysis — `read_rom_bytes` variant), §7 (Hardware — P0 tools).

See `docs/architecture.md` for the tool implementation pattern.

## Acceptance Criteria

- [ ] `gb_read_memory` — Reads up to 4096 bytes from any memory address (RAM, VRAM, I/O), returns hexdump + ASCII via `utils/hexdump`. Validates length (1-4096) and address range. Returns `{"error": "NO_ROM_LOADED", ...}` if no ROM loaded.
- [ ] `gb_read_banked` — Reads memory from a specific bank using `pyboy.memory[bank, address]`, returns hexdump. Validates bank number. Returns error if no ROM loaded.
- [ ] `gb_get_bank_info` — Returns MBC type (via `domain/bank_info`), current ROM/RAM bank, total bank count. Reads cartridge header for MBC state. Returns error if no ROM loaded.
- [ ] `gb_get_interrupt_status` — Reads IE (0xFFFF) and IF (0xFF0F) from emulator memory, delegates to `domain/interrupts` for flag interpretation. Returns structured JSON with individual flag states. Returns error if no ROM loaded.
- [ ] All 4 tools are registered in `server.py` and accessible via MCP.
- [ ] Integration tests cover happy path and error paths (no ROM loaded) for all 4 tools.

## Technical Approach

### Files

**`src/blobert_mcp/tools/memory.py`**
- `gb_read_memory(address: int, length: int = 256) -> dict`
- `gb_read_banked(bank: int, address: int, length: int = 256) -> dict`
- `gb_get_bank_info() -> dict`
- `gb_get_interrupt_status() -> dict`

**`src/blobert_mcp/server.py`** — Import and register memory tool module.

### Tool implementation pattern

Same 5-step recipe as BLO-013:
1. Validate inputs
2. Check preconditions (ROM loaded)
3. Read raw bytes from emulator
4. Delegate to domain for interpretation
5. Format and return

### Integration tests

Test setup uses `EmulatorSession.load_rom()` directly (from BLO-001's `emulator.py`) — no dependency on BLO-013's MCP tool wrappers. This keeps the dependency honest: BLO-014 depends on BLO-012 (domain layer) and BLO-001 (emulator), not on BLO-013.

## Notes

- `gb_read_memory` reads flat memory addresses. For banked regions (ROMX, WRAMX on CGB), the current bank is used. Use `gb_read_banked` for explicit bank selection.
- `gb_get_bank_info` current bank detection is MBC-type-dependent. For MBC1: read 0x2000-0x3FFF region. For MBC3/MBC5: similar but with different register layouts. Start simple and iterate.
- This ticket can be worked in parallel with BLO-013 after BLO-012 completes, since it has no dependency on the session/static tools.
