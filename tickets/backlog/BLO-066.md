# BLO-066: PyPI package publishing

## Objective

Publish blobert-mcp to PyPI so users can install it with `pip install blobert-mcp`. This includes setting up a GitHub Actions workflow triggered on tag push (v*), building with `uv build`, and publishing via trusted publisher (OIDC). The definition of done is a successful `pip install blobert-mcp && blobert-mcp --help` from a clean environment.

## Context

The project is currently only installable from source. Publishing to PyPI makes it accessible to the broader community and enables simple installation for end users. The project already has its entry point configured in pyproject.toml as "blobert-mcp", so the package metadata foundation exists.

## Acceptance Criteria

- [ ] Package metadata in pyproject.toml is complete and valid for PyPI (description, license, classifiers, URLs, etc.)
- [ ] GitHub Actions workflow (.github/workflows/publish.yml) triggers on tag push matching `v*`
- [ ] Workflow uses `uv build` to create wheel and sdist
- [ ] Workflow uses trusted publisher (OIDC) to upload to PyPI — no API tokens stored in secrets
- [ ] PyPI project is registered and configured for trusted publishing from the repository
- [ ] `pip install blobert-mcp && blobert-mcp --help` succeeds from a clean environment

## Technical Approach

1. Review and update `pyproject.toml` to ensure all required PyPI metadata fields are present (description, author, license, classifiers, project URLs, readme).
2. Create `.github/workflows/publish.yml`:
   - Trigger on `push: tags: ['v*']`
   - Job: build sdist + wheel using `uv build`
   - Job: publish to PyPI using `pypa/gh-action-pypi-publish` with trusted publishing (OIDC, no token needed)
   - Use `id-token: write` permission for OIDC
3. Register the project on PyPI and configure trusted publisher (GitHub Actions, repository, workflow file).
4. Test with a pre-release tag (e.g., `v0.1.0rc1`) to validate the pipeline end-to-end.

## Notes

- PyPI trusted publishing (OIDC) eliminates the need for long-lived API tokens. See https://docs.pypi.org/trusted-publishers/
- The PyPI project must be registered manually before the first automated publish.
- Consider adding a TestPyPI step for pre-release validation.
