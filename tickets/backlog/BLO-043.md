# BLO-043: Visual diff and entropy analysis

## Objective

Implement two analysis tools: `gb_visual_diff` for comparing screenshots from two save states and returning a pixel-difference heatmap as an MCP Image, and `calculate_entropy` for computing Shannon entropy over a byte range to distinguish code, compressed data, and padding. This ticket is done when both tools are registered, tested, and return correct results.

## Context

The spec defines `gb_visual_diff` (section 6, P2) and `calculate_entropy` (section 1, P2) as analysis tools. `gb_visual_diff` builds on the existing `gb_screenshot` infrastructure (PIL + MCP Image, implemented in BLO-022). `calculate_entropy` is pure math on ROM bytes. Both tools help the LLM make better decompilation decisions -- visual diff shows what changed on screen after an action, and entropy analysis reveals the nature of data regions.

See `docs/mcp-spec.md` sections 1 and 6 for tool definitions.

## Acceptance Criteria

- [ ] `gb_visual_diff(state_id_a, state_id_b)` tool is registered and callable via MCP
- [ ] `gb_visual_diff` loads screenshots from two save states, computes pixel differences, and returns a heatmap image as MCP Image (base64)
- [ ] `gb_visual_diff` returns a clear error if either state_id is invalid
- [ ] `calculate_entropy(address, length)` tool is registered and callable via MCP
- [ ] `calculate_entropy` returns a list of Shannon entropy scores (0.0 to 1.0) per window over the specified byte range
- [ ] `calculate_entropy` accepts an optional `window_size` parameter (default 256)
- [ ] Domain module `domain/analysis.py` contains the entropy calculation logic (pure function, stdlib only)
- [ ] Unit tests exist for entropy calculation (TDD) -- test with known byte patterns (all zeros = 0.0, uniform random ~ 1.0)
- [ ] Unit tests exist for visual diff logic (TDD)
- [ ] Tool handler tests exist for both tools

## Technical Approach

1. **`domain/analysis.py`**: New domain module containing:
   - `calculate_shannon_entropy(data: bytes, window_size: int) -> list[float]`: Slide a window over the data, compute Shannon entropy for each window. Uses `math.log2` and byte frequency counting. Pure function, no external dependencies.
   - `compute_pixel_diff(image_a: PIL.Image, image_b: PIL.Image) -> PIL.Image`: Compute per-pixel absolute difference and generate a heatmap image. Uses PIL for image manipulation.
2. **`gb_visual_diff` tool**: Load save states by ID, capture screenshots from each (using existing screenshot infrastructure), pass to domain function, return heatmap as MCP Image (base64-encoded PNG).
3. **`calculate_entropy` tool**: Read bytes from ROM at the given address/length, pass to domain entropy function, return list of entropy scores with metadata.
4. **Tests**: Write domain tests first (TDD). Entropy tests use synthetic byte arrays with known entropy values. Visual diff tests use small synthetic PIL images.

## Notes

- Entropy calculation uses only `math` (stdlib) -- no numpy dependency needed.
- PIL is already a project dependency (used by `gb_screenshot`).
- The heatmap could use a simple grayscale (brighter = more different) or a color gradient. Start with grayscale for simplicity.
- High entropy (~1.0) suggests compressed/encrypted data. Low entropy (~0.0) suggests padding/unused space. Medium entropy suggests code or text.
