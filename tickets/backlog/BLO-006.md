# BLO-006: Repository foundation files

## Objective

Add the essential repository configuration files (`.gitignore`, `LICENSE`, `.python-version`, `.editorconfig`) that must exist before any source code is committed. After this ticket, the repository is safe from accidental artifact commits and legally unambiguous.

## Context

The README badge claims MIT license but no `LICENSE` file exists. There is no `.gitignore`, meaning BLO-001 work risks committing `__pycache__/`, `.venv/`, ROM binaries, and IDE files. These foundation files have zero code dependencies and should be completed before BLO-001.

## Acceptance Criteria

- [ ] `.gitignore` exists with rules for: Python bytecode (`__pycache__/`, `*.pyc`, `*.pyo`), virtual environments (`.venv/`, `venv/`, `env/`), distribution/packaging (`dist/`, `build/`, `*.egg-info/`), IDE files (`.idea/`, `.vscode/`, `*.swp`), OS files (`.DS_Store`, `Thumbs.db`), testing (`.pytest_cache/`, `.coverage`, `htmlcov/`), Game Boy ROMs (`*.gb`, `*.gbc`, `*.sgb`), and database files (`*.db`)
- [ ] `LICENSE` contains the full MIT license text with copyright year 2026 and the repository owner. The README badge `[![License: MIT](...)](LICENSE)` resolves correctly.
- [ ] `.python-version` contains `3.10` (matching the Python 3.10+ constraint in README and CLAUDE.md)
- [ ] `.editorconfig` specifies: UTF-8 charset, LF line endings, 4-space indent for Python, 2-space indent for YAML/JSON/TOML, trailing whitespace trimming, final newline insertion

## Technical Approach

1. **`.gitignore`** — Start from GitHub's Python template. Add Game Boy ROM extensions (`.gb`, `.gbc`, `.sgb`), PyBoy-specific patterns, blobert-mcp knowledge base database files (`*.db`), and `uv.lock` exclusion is NOT needed (lock files should be committed).

2. **`LICENSE`** — Standard MIT license text. Copyright holder from git remote URL (`kdridi`).

3. **`.python-version`** — Single line: `3.10`. Enables `pyenv` and `uv` to auto-detect the required Python version.

4. **`.editorconfig`** — Root editorconfig with sections for `[*]`, `[*.py]`, `[*.{yml,yaml,json,toml}]`, `[*.md]`, `[Makefile]`.
