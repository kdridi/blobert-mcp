# BLO-012: Domain modules & utility foundation (TDD)

## Objective

Implement all five domain modules and the hexdump utility using strict TDD — tests written before implementation, synthetic byte inputs only, zero external dependencies. After this ticket, the pure computation layer is complete: ROM header parsing, memory layout definitions, MBC bank detection, interrupt flag interpretation, vector table data, and hex formatting are all tested and ready for the tool layer to consume.

## Context

This is the first of three tickets decomposed from BLO-002. It implements the bottom layer of the architecture (`domain/` + `utils/`) in isolation, following the dependency rule that domain modules import nothing from the project. By completing this layer first, the tool tickets (BLO-013, BLO-014) can focus purely on MCP wiring without mixing concerns.

Covers the domain logic for spec sections: §1 (Static analysis), §7 (Hardware), §9 (Session).

See `docs/architecture.md` for layer rules and `docs/testing-strategy.md` for TDD workflow.

## Acceptance Criteria

- [ ] Directory structure exists: `src/blobert_mcp/domain/`, `src/blobert_mcp/utils/`, `tests/domain/`, `tests/utils/` — each with `__init__.py`.
- [ ] `domain/rom_header.py` — `parse(data: bytes) -> dict` parses 0x0100-0x014F header bytes. Returns title, cartridge type, ROM size, RAM size, licensee, CGB flag, SGB flag, header checksum, global checksum. Tests use synthetic byte sequences.
- [ ] `domain/memory_map.py` — Provides the static Game Boy memory layout: ROM0, ROMX, VRAM, SRAM, WRAM0, WRAMX, Echo, OAM, Unusable, I/O, HRAM, IE register — each with name, start address, end address, size, and access type. Tests verify all regions with correct addresses.
- [ ] `domain/bank_info.py` — `detect_mbc_type(cartridge_byte: int) -> dict` returns MBC type name and capabilities. `calculate_bank_count(rom_size_byte: int) -> int` returns total ROM bank count. Covers ROM-only, MBC1, MBC3, MBC5. Tests cover all common cartridge type bytes.
- [ ] `domain/interrupts.py` — `parse_interrupt_flags(ie_byte: int, if_byte: int) -> dict` interprets IE/IF registers into individual flag states (VBlank, STAT, Timer, Serial, Joypad). Tests verify all 5 flags with various bit patterns.
- [ ] `domain/vectors.py` — Defines RST vector addresses (0x00, 0x08, 0x10, 0x18, 0x20, 0x28, 0x30, 0x38), interrupt vector addresses (0x40, 0x48, 0x50, 0x58, 0x60), and entry point (0x100) as structured data. Tests verify all addresses and labels.
- [ ] `utils/hexdump.py` — `hexdump(data: bytes, start_offset: int = 0) -> str` formats bytes as offset + hex (16 per line, space-separated) + ASCII sidebar. Tests verify formatting with full lines, partial lines, empty input, and non-printable characters.
- [ ] All domain modules import nothing from the project — only Python stdlib and `dataclasses`. Verified by inspection.

## Technical Approach

### TDD workflow (mandatory for every module)

1. Create the test file first (e.g., `tests/domain/test_rom_header.py`)
2. Write test cases with known byte inputs and expected structured outputs
3. Run the test — it fails (module does not exist yet)
4. Create the domain module (e.g., `src/blobert_mcp/domain/rom_header.py`)
5. Implement until tests pass
6. Refactor if needed — tests must stay green

### Domain modules

**`domain/rom_header.py`** — Parse ROM header bytes (0x0100-0x014F):
- 0x0134-0x0143: Title (ASCII, null-terminated)
- 0x0147: Cartridge type (MBC)
- 0x0148: ROM size
- 0x0149: RAM size
- 0x014B: Old licensee code
- 0x0143: CGB flag
- 0x0146: SGB flag
- 0x014D: Header checksum
- 0x014E-0x014F: Global checksum

**`domain/memory_map.py`** — Static definitions, no computation needed. List of memory region dataclasses.

**`domain/bank_info.py`** — Two pure functions:
- `detect_mbc_type(cartridge_byte)` — Lookup table from cartridge type byte to MBC info
- `calculate_bank_count(rom_size_byte)` — `2 << rom_size_byte` for standard sizes

**`domain/interrupts.py`** — Bit masking on IE (0xFFFF) and IF (0xFF0F) registers:
- Bit 0: VBlank, Bit 1: STAT, Bit 2: Timer, Bit 3: Serial, Bit 4: Joypad

**`domain/vectors.py`** — Static data: vector addresses and labels as a list of dataclasses.

### Utils

**`utils/hexdump.py`** — Format: `OFFSET  HH HH HH ... HH  |ASCII...........|`
- 16 bytes per line, space-separated hex
- Non-printable chars shown as `.` in ASCII column
- Configurable start offset for correct address display
