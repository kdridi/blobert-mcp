# BLO-021: Core dynamic tools

## Objective

Implement the 6 MCP tools that form the complete interactive exploration loop: stepping through frames, running to an address, reading registers, pressing buttons, and managing save states. After this ticket, an LLM can load a ROM, advance execution, press buttons, checkpoint/restore state, and read CPU state — the full dynamic workflow minus instruction-level stepping and screenshots.

## Context

This is the second of three tickets decomposed from BLO-004 (rejected). It builds the tool layer on top of the domain modules from BLO-020 (`domain/registers.py`, `domain/buttons.py`). These 6 tools use well-known PyBoy APIs with no significant unknowns.

Covers spec sections: §2 (Dynamic execution — P0 tools, frame mode only), §4 (Save states — P0 tools), §5 (Inputs — P0 tool).

## Acceptance Criteria

- [ ] `gb_step` — Advances execution by `count` frames (mode="frame"). Returns current PC, current instruction (mnemonic + operands), and full register state via `domain/registers.format_registers()`. Uses `pyboy.tick()`.
- [ ] `gb_run_until` — Executes until `target_address` is hit or `timeout_frames` exceeded (default 600). Uses `pyboy.hook_register()`. Returns registers at breakpoint or `{"error": "TIMEOUT", ...}`.
- [ ] `gb_get_registers` — Returns all CPU registers in spec format via `domain/registers.format_registers()`. Reads from `pyboy.register_file`.
- [ ] `gb_press_button` — Presses or releases a button. Validates via `domain/buttons.validate_button()` and `domain/buttons.validate_action()`. Uses `pyboy.button_press()`/`pyboy.button_release()`.
- [ ] `gb_save_state` — Saves emulator state to in-memory BytesIO. Stores in `session.save_states` dict with auto-incrementing ID. Supports optional `name`. Returns state_id + metadata.
- [ ] `gb_load_state` — Restores a previously saved state by state_id. Returns registers after restoration. Returns `{"error": "NOT_FOUND", ...}` for invalid state_id.
- [ ] `EmulatorSession` has `save_states: dict` and `next_state_id: int` fields initialized in `__init__`.
- [ ] All 6 tools registered in `server.py`.
- [ ] All tools return `{"error": "NO_ROM_LOADED", ...}` when no ROM is loaded.
- [ ] Integration tests cover happy paths and error paths.

## Technical Approach

### Tool files (following 5-step recipe from `docs/architecture.md`)

**`src/blobert_mcp/tools/execution.py`** — `register_execution_tools(mcp, session)`:
- `gb_step(count: int = 1, mode: str = "frame") -> dict` — frame mode only in this ticket. Validate mode is "frame" (return error for "instruction" — deferred to BLO-022).
- `gb_run_until(target_address: int, timeout_frames: int = 600) -> dict` — hook + tick loop.
- `gb_get_registers() -> dict` — read register_file, delegate to `format_registers()`.

**`src/blobert_mcp/tools/input.py`** — `register_input_tools(mcp, session)`:
- `gb_press_button(button: str, action: str = "press") -> dict` — validate, call PyBoy.

**`src/blobert_mcp/tools/savestate.py`** — `register_savestate_tools(mcp, session)`:
- `gb_save_state(name: str | None = None) -> dict` — BytesIO save, store in session.
- `gb_load_state(state_id: int) -> dict` — restore, return registers.

### Emulator changes

**`src/blobert_mcp/emulator.py`** — Add to `EmulatorSession.__init__`:
```python
self.save_states: dict[int, dict] = {}
self.next_state_id: int = 1
```

### Server registration

**`src/blobert_mcp/server.py`** — Import and call `register_execution_tools`, `register_input_tools`, `register_savestate_tools`.

### Register reading pattern

All tools that return registers use this pattern:
```python
rf = session.pyboy.register_file
result = registers.format_registers(
    rf.A, rf.B, rf.C, rf.D, rf.E, rf.F, rf.H, rf.L, rf.SP, rf.PC
)
```

## Notes

- `gb_step` mode="instruction" is intentionally deferred to BLO-022. This ticket returns `{"error": "NOT_IMPLEMENTED", "message": "Instruction stepping is not yet available. Use mode='frame'."}` for that mode.
- PyBoy's `tick()` advances one frame (~16.7ms of Game Boy time, ~70224 CPU cycles).
- For `gb_run_until`, the hook callback should set a flag; the tick loop checks the flag after each tick.
- Save states via BytesIO keep everything in memory. States are lost when the server stops. Acceptable for V1.
- `gb_press_button` does NOT auto-step — the LLM controls timing.
- This ticket mirrors BLO-013 (session & static tools) in structure — 6 tools across 3 modules.
