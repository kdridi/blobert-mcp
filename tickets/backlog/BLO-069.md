# BLO-069: Example workflows (cookbook)

## Objective

Create a collection of tutorial-style guides in `docs/cookbook/` that demonstrate real decompilation workflows using blobert-mcp's MCP tools. Each guide walks through a concrete scenario step-by-step, showing exact tool calls and expected outputs, so users can learn by example.

## Context

With 25 MCP tools implemented, new users need practical guidance on how to combine tools into effective decompilation workflows. Abstract API documentation is not enough — users benefit from seeing complete, realistic examples that show the tools working together on actual ROM analysis tasks. A cookbook bridges the gap between "I know what tools exist" and "I know how to use them."

## Acceptance Criteria

- [ ] `docs/cookbook/` directory created
- [ ] Guide: "Analyzing a ROM header" — demonstrates loading a ROM and inspecting header fields (title, cartridge type, ROM/RAM size, checksums)
- [ ] Guide: "Tracing a game's battle system" — demonstrates using execution tools to step through combat logic, setting breakpoints, and annotating discovered routines
- [ ] Guide: "Annotating a simple homebrew game" — demonstrates a full annotation workflow: disassemble, label entry points, add comments, save to KB
- [ ] Guide: "Finding and labeling string tables" — demonstrates searching ROM data for text strings, identifying table boundaries, and creating KB annotations
- [ ] Each guide is a standalone `.md` file with a clear title, prerequisites, and numbered steps
- [ ] Each step shows the exact MCP tool call (with parameters) and the expected output format
- [ ] Guides use synthetic or public-domain ROM examples (no copyrighted material)
- [ ] A `docs/cookbook/README.md` index file lists all available guides with one-line descriptions

## Technical Approach

- Create `docs/cookbook/` directory with an index `README.md`.
- Each guide follows a consistent structure: title, overview, prerequisites (which tools/state are needed), step-by-step walkthrough, summary.
- Tool calls should be shown in a code block with JSON parameters, followed by a description of the expected output.
- Use synthetic byte sequences or public-domain homebrew ROMs for examples. Never reference copyrighted ROM files.
- Keep each guide focused on one workflow — avoid trying to cover everything in a single document.
- Cross-reference the API documentation and architecture docs where relevant.
