# BLO-057: Documentation generator from KB

## Objective

Implement a `kb_generate_docs` MCP tool that generates navigable documentation from Knowledge Base annotations. The tool produces either a directory of Markdown files or a static HTML site, giving users a browsable reference of all annotated functions, variables, and cross-references stored in the KB. Done when both output formats generate correct, linked documentation from KB data.

## Context

With 25 MCP tools implemented and a SQLite-backed KB storing annotations, functions, and variables, users have no way to export that accumulated knowledge into a human-readable, navigable format. A documentation generator closes this gap, making the decompilation work shareable and reviewable outside the MCP tool interface. Depends on the KB infrastructure (BLO-005) and KB query capabilities (BLO-016).

## Acceptance Criteria

- [ ] New domain module `domain/doc_generator.py` with pure functions for generating documentation structures
- [ ] `kb_generate_docs(format?, output_dir?)` MCP tool registered and functional
- [ ] `format` parameter accepts `"markdown"` (default) and `"html"`
- [ ] Markdown output: directory of `.md` files with index page, function list, and per-function pages
- [ ] HTML output: static site with the same structure, navigable in a browser
- [ ] Per-function pages include disassembly, variables, and cross-references from the KB
- [ ] Index page links to all function pages
- [ ] Tests written first (TDD) using synthetic KB data — no real ROMs
- [ ] Tool returns `{"error": "CODE", "message": "..."}` on failure (e.g., empty KB, invalid format)

## Technical Approach

1. **Domain module** (`src/blobert_mcp/domain/doc_generator.py`):
   - Pure functions that take KB data structures and produce documentation content
   - `generate_index(functions) -> str` — renders the index/table of contents
   - `generate_function_page(func, variables, xrefs, disassembly) -> str` — renders a single function page
   - `generate_docs(kb_data, format) -> dict[str, str]` — orchestrates generation, returns filename-to-content mapping
   - Separate rendering logic for Markdown vs HTML

2. **Tool handler** (`src/blobert_mcp/tools/`):
   - Validate parameters (format, output_dir)
   - Query KB for all functions, variables, cross-references
   - Delegate to domain module
   - Write files to output_dir
   - Return summary (file count, output path)

3. **Testing**:
   - Domain tests with synthetic KB data (function dicts, variable lists)
   - Tool tests verifying parameter validation and file output
   - Verify cross-reference links resolve correctly in both formats

## Dependencies

- BLO-005: KB infrastructure (SQLite storage)
- BLO-016: KB query capabilities

- Cross-reference links between function pages should use relative paths for portability
- HTML output could include a minimal CSS stylesheet for readability
