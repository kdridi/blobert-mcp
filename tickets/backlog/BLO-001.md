# BLO-001: Project infrastructure & MCP server skeleton

## Objective

Set up the Python project structure, dependencies, and a minimal MCP server that starts, connects, and registers tool stubs. After this ticket, `uv run blobert-mcp` launches a working MCP server with no tools yet ‚Äî but the wiring is complete and ready for tools to be added incrementally.

## Context

The project has a complete spec (`docs/mcp-spec.md`) and research (`docs/journal.md`) but zero source code. Every other ticket depends on this foundation: package config, project layout, MCP server entry point, and the emulator session wrapper that all tools will share.

This is the first ticket ‚Äî it unblocks everything else.

## Acceptance Criteria

- [ ] `pyproject.toml` exists with project metadata, Python >=3.10, and dependencies: `pyboy`, `mcp` (MCP SDK), `Pillow`
- [ ] Project uses `uv` for package management; `uv sync` installs all dependencies
- [ ] Source layout: `src/blobert_mcp/` package with `__init__.py`, `server.py`, `emulator.py`
- [ ] `server.py` creates an MCP server instance, registers a health-check tool (`ping`), and exposes stdio transport
- [ ] `emulator.py` contains an `EmulatorSession` class that wraps PyBoy lifecycle (init, load ROM, shutdown) ‚Äî no tools yet, just the shared state object
- [ ] Entry point configured: `uv run blobert-mcp` starts the MCP server
- [ ] The server starts without errors when no ROM is loaded (graceful empty state)
- [ ] Basic `tests/` directory with a smoke test that imports the package

## Technical Approach

1. **`pyproject.toml`** ‚Äî Use `[project]` table with `[project.scripts]` for the entry point. Pin `pyboy>=2.0.0`, `mcp>=1.0.0`, `Pillow>=10.0.0`. Dev dependencies: `pytest`.

2. **`src/blobert_mcp/__init__.py`** ‚Äî Package marker, export version.

3. **`src/blobert_mcp/emulator.py`** ‚Äî `EmulatorSession` class:
   - Holds the `PyBoy` instance (nullable ‚Äî None when no ROM loaded)
   - `load_rom(path, headless=True)` ‚Äî creates PyBoy instance
   - `shutdown()` ‚Äî cleans up
   - Holds shared state: save states dict, breakpoints dict, knowledge base reference
   - Singleton-ish: one session per server process

4. **`src/blobert_mcp/server.py`** ‚Äî MCP server setup:
   - Create `Server` instance from MCP SDK
   - Register `ping` tool (returns `{"status": "ok", "rom_loaded": bool}`)
   - `main()` function that runs the stdio transport
   - The `EmulatorSession` instance is created here and passed to tool handlers via closure or context

5. **Entry point** ‚Äî `[project.scripts] blobert-mcp = "blobert_mcp.server:main"`

6. **`tests/test_smoke.py`** ‚Äî Import check, verify server can be instantiated.
