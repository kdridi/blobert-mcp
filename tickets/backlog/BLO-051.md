# BLO-051: ROM comparison tool

## Objective

Implement `gb_compare_roms(rom_path_a, rom_path_b, region?)` — a tool that performs binary diff between two Game Boy ROM files and returns a list of difference regions. This enables users to compare original vs translated ROMs, detect patches, and find modified routines.

## Context

When working with ROM hacking, translation patches, or reverse engineering, it is essential to compare two ROM versions to identify what changed. Currently there is no way to diff two ROMs through the MCP interface. This tool fills that gap by providing structured difference output with optional region filtering (header, specific banks, etc.).

## Acceptance Criteria

- [ ] `gb_compare_roms` MCP tool registered and callable
- [ ] Accepts `rom_path_a` and `rom_path_b` as required parameters
- [ ] Accepts optional `region` parameter to filter by ROM region (e.g., "header", "bank:N")
- [ ] Returns list of `{offset, length, bytes_a, bytes_b}` difference regions
- [ ] Consecutive differing bytes are coalesced into single regions
- [ ] Region filtering correctly restricts comparison to header (0x0000-0x014F) or specific bank boundaries
- [ ] Returns meaningful error when files don't exist or aren't valid ROM sizes
- [ ] Domain module `domain/rom_diff.py` contains all comparison logic (no business logic in tool handler)
- [ ] Unit tests for domain module written before implementation (TDD)
- [ ] Tool handler in `tools/` delegates to domain, validates inputs, formats output

## Technical Approach

1. **Domain module (`domain/rom_diff.py`)**:
   - `diff_roms(bytes_a: bytes, bytes_b: bytes, start: int = 0, end: int | None = None) -> list[DiffRegion]`
   - `DiffRegion` dataclass: `offset`, `length`, `bytes_a`, `bytes_b`
   - Coalesce consecutive differing bytes into contiguous regions
   - Region-to-offset-range mapping for header, bank boundaries (16 KiB banks)

2. **Tool handler (`tools/gb_compare_roms.py`)**:
   - Validate both paths exist and are readable
   - Load ROM bytes, parse optional `region` parameter into offset range
   - Delegate to `diff_roms`, format result as JSON-friendly output

3. **Tests (`tests/domain/test_rom_diff.py`)**:
   - Identical ROMs produce empty diff
   - Single-byte difference
   - Consecutive differences coalesced into one region
   - Region filtering limits scope
   - Mismatched ROM sizes handled gracefully
