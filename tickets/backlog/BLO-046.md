# BLO-046: Session utility tools

## Objective

Implement two lightweight session utility MCP tools: `gb_get_frame_count()` to return the number of frames elapsed since ROM load, and `gb_set_speed(multiplier)` to change the emulation speed. Done when both tools are registered, functional, and have full test coverage.

## Context

Frame count and emulation speed are simple but useful session utilities. `gb_get_frame_count` lets the LLM track temporal position within an emulation session (e.g., "how far into the game are we?"), which is especially valuable after context window truncation when the LLM loses track of prior stepping. `gb_set_speed` enables the LLM to speed up or slow down emulation for different tasks -- unlimited speed for fast-forwarding through menus, normal speed for frame-accurate analysis. Both are thin wrappers around existing PyBoy APIs. See `docs/mcp-spec.md` section 9: `gb_get_frame_count` (P1) and `gb_set_speed` (P2).

## Acceptance Criteria

- [ ] `gb_get_frame_count()` tool is registered as an MCP tool with no required parameters
- [ ] Returns `{"frame_count": int}` with the number of frames elapsed since the ROM was loaded
- [ ] `gb_set_speed(multiplier)` tool is registered as an MCP tool
- [ ] Accepts `multiplier` parameter: 0 = unlimited, 1 = normal (60fps), 2 = double, etc.
- [ ] Returns confirmation with the new speed setting
- [ ] Validates multiplier is a non-negative integer
- [ ] Both tools return `{"error": "NO_ROM_LOADED", ...}` when no ROM is loaded
- [ ] Domain logic (if any) is TDD -- tests written before implementation
- [ ] Tool handlers are thin: validate, call PyBoy API, format output

## Technical Approach

1. **`gb_get_frame_count`** -- Direct PyBoy API wrapper:
   - Read `pyboy.frame_count` (the frame counter is already tracked by the session via PyBoy)
   - Return as JSON `{"frame_count": N}`
2. **`gb_set_speed`** -- Direct PyBoy API wrapper:
   - Call `pyboy.set_emulation_speed(multiplier)`
   - Store the current speed setting in EmulatorSession for reporting in `get_session_info`
   - Return confirmation `{"multiplier": N, "description": "unlimited"|"normal"|"Nx"}`
3. **Tool handlers** -- Register both tools following existing patterns. Minimal domain logic needed since these are direct API calls; validation is the main domain concern.
4. **Tests** -- Unit tests for input validation. Integration tests verifying PyBoy API calls with mock emulator.

## Notes

- `pyboy.frame_count` is available as a property on the PyBoy instance. Verify it resets on ROM load (it should, since a new PyBoy instance is created).
- `gb_get_frame_count` overlaps with the `frame_count` field already returned by `get_session_info`. The dedicated tool is useful when only the frame count is needed without the overhead of the full session info response.
- Consider whether `gb_set_speed(0)` (unlimited) should emit a warning about CPU usage in headless mode.
