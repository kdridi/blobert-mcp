# BLO-056: Symbol file auto-import for known projects

## Objective

Automatically detect well-known Game Boy ROMs on load and import their community-maintained symbol files, giving the LLM immediate access to meaningful labels instead of raw addresses. Definition of done: when a known ROM is loaded, its .sym file is fetched and imported into the KB without user intervention.

## Context

Several well-known Game Boy decompilation projects (pokered, pokecrystal, Tetris, etc.) publish `.sym` symbol files that map addresses to human-readable labels. Manually importing these is tedious and error-prone. By maintaining a built-in database of game signatures (title + checksum), the server can recognize known ROMs and automatically pull in the correct symbols, dramatically improving the decompilation experience from the first interaction.

## Acceptance Criteria

- [ ] Built-in game signatures database maps `{title, checksum}` to `{repo_url, sym_path, format}`
- [ ] On ROM load, title and checksum are checked against the signatures database
- [ ] If a match is found, the corresponding .sym file is fetched from the community repository
- [ ] Fetched symbols are imported into the KB using the existing BLO-040 import logic
- [ ] Auto-import can be disabled via a configuration flag or tool parameter
- [ ] Import result is reported to the user (number of symbols imported, source URL)
- [ ] Graceful handling when fetch fails (network error, repo unavailable) — ROM still loads
- [ ] At least 2 known games are included in the signatures database (Pokemon Red, Pokemon Crystal)
- [ ] Domain logic for signature matching and import orchestration is covered by tests (TDD)
- [ ] Symbol file format parsing handles the rgbds `.sym` format used by pokered/pokecrystal

## Technical Approach

1. **Domain layer** (`src/blobert_mcp/domain/symbols/`):
   - `GameSignature` dataclass: `title`, `checksum`, `repo_url`, `sym_path`, `format`.
   - `KNOWN_GAMES` registry: a list/dict of `GameSignature` entries.
   - `match_rom(title, checksum) -> Optional[GameSignature]`: lookup function.
   - `parse_sym_file(content: str, format: str) -> list[Symbol]`: parse .sym file content into structured symbols.
   - Symbol format: rgbds `.sym` files use `BANK:ADDR label` format (e.g., `00:0150 Start`).

2. **Fetch layer** (`src/blobert_mcp/domain/symbols/` or infrastructure):
   - Fetch .sym file from GitHub raw content URL.
   - Cache fetched files locally to avoid repeated downloads.
   - Handle network errors gracefully.

3. **Integration with ROM load**:
   - After ROM is loaded and header is parsed, call `match_rom()`.
   - If match found and auto-import enabled, fetch and import symbols via BLO-040 KB import.
   - Log the result (symbols imported count, source).

4. **Initial game database**:
   - Pokemon Red (US): title `POKEMON RED`, checksum matching, pokered repo .sym file.
   - Pokemon Crystal (US): title `POKEMON CRYSTAL`, checksum matching, pokecrystal repo .sym file.

## Notes

- The `.sym` format from rgbds is `BANK:ADDR label` per line, with optional comments starting with `;`.
- Community repos: `pret/pokered`, `pret/pokecrystal` on GitHub.
- Consider supporting `.map` files in addition to `.sym` in a future ticket.
- The signatures database should be easy to extend — adding a new game should be a single entry addition.
- Network access during ROM load may be unexpected for users — the disable flag is important.
