# BLO-058: Test ROM generator for CI

## Objective

Create a Python utility that programmatically assembles SM83 instructions into minimal valid Game Boy ROM files for integration testing. This enables end-to-end testing of disassembly, analysis, and KB tools without relying on copyrighted ROMs. Done when the builder can produce at least four distinct test ROMs that pass header validation and exercise different CPU/memory features.

## Context

The project currently uses synthetic byte sequences for unit tests (459 tests passing), but lacks valid ROM files for integration and end-to-end testing. Real Game Boy ROMs are copyrighted and cannot be included in the repository. A ROM builder that generates minimal but structurally valid ROMs solves this by creating test fixtures programmatically. Depends on BLO-013 (ROM loading/header parsing infrastructure).

## Acceptance Criteria

- [ ] Python module/script for assembling SM83 instructions into valid Game Boy ROM files
- [ ] Generated ROMs have correct Game Boy header (Nintendo logo, checksum, cartridge type)
- [ ] Test ROM: `nop_loop` — NOP sled that loops back to start
- [ ] Test ROM: `counter` — increments a RAM variable in a loop
- [ ] Test ROM: `hello` — writes characters to VRAM/screen area
- [ ] Test ROM: `banking` — exercises MBC1 bank switching
- [ ] ROMs placed in `tests/fixtures/` or `tests/rom_builder/`
- [ ] At least one integration test that loads a generated ROM through the MCP tools
- [ ] Tests written first (TDD) — builder output validated against known byte patterns
- [ ] ROM builder usable as both a library (import) and a CLI script

## Technical Approach

1. **ROM builder module** (`tests/rom_builder/`):
   - `rom_builder.py` — core builder class
   - `RomBuilder` class with methods: `add_instruction(opcode, operands)`, `set_entry_point(addr)`, `set_cartridge_type(type)`, `build() -> bytes`
   - Header generation: Nintendo logo bytes, title, cartridge type, ROM size, checksums
   - SM83 instruction encoding for common instructions (NOP, LD, INC, JR, CALL, RET, etc.)
   - Bank management for MBC1 ROMs

2. **Predefined test ROMs** (`tests/rom_builder/test_roms.py`):
   - Factory functions: `make_nop_loop()`, `make_counter()`, `make_hello()`, `make_banking()`
   - Each returns valid ROM bytes
   - Can optionally write to disk as `.gb` files

3. **Fixtures integration** (`tests/fixtures/`):
   - Generated `.gb` files committed or generated at test time via `conftest.py`
   - Pytest fixtures that provide ROM bytes to integration tests

4. **Testing**:
   - Unit tests for instruction encoding (known opcode → expected bytes)
   - Unit tests for header generation (checksums, logo bytes)
   - Integration tests loading generated ROMs through existing tools

## Notes

- The Nintendo logo bytes (0x0104-0x0133) are required for header validity — these are well-documented fixed bytes
- Keep the instruction set minimal: only encode the SM83 instructions actually needed by the test ROMs
- Consider whether to commit generated `.gb` files or regenerate them during test setup
- The `banking` ROM needs at least 2 banks (64KB) to exercise MBC1 switching
